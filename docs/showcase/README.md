>
> 「実物（既に動いているもの）」「設計サンプル」「未実装の計画」を **混同せず提示する** ことを意識しています。

## 4. インシデントタイムライン

[07. インシデント対応](../server-monitor-improvements/07-incident-response.md) 計画書の運用イメージ。

```mermaid
flowchart TD
    A[14:02 検知<br/>Alertmanager Slack 通知] --> B[14:05 Sev2 宣言<br/>IC アサイン]
    B --> C[14:10 一次仮説<br/>Grafana 再起動]
    C --> D[14:25 切り分け<br/>504 ログ多発]
    D --> E[14:40 仮説変更<br/>Nginx upstream 設定]
    E --> F[14:48 設定変更<br/>keepalive 75s]
    F --> G[14:52 復旧宣言<br/>プローブ全成功]
    G --> H[14:55 周知<br/>状況サマリ Slack]
    H --> I[翌営業日<br/>ポストモーテム]
```
