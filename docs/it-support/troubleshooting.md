# トラブルシューティングフロー集

> **本ドキュメントの位置付け**
>
> [FAQ](./faq.md) と同様、IT サポート業務での切り分け設計を示す **サンプル / テンプレート** です。実際の現場対応記録ではありません。

代表的な障害について、**「現場で迷わない切り分け順序」** を Mermaid フローチャートで整理しています。
FAQ より一段深く、判断分岐を持った形で記述しています。

## 1. インターネットに繋がらない

```mermaid
flowchart TD
    A[ユーザー報告: インターネット繋がらない] --> B{他の社員も<br/>同じ症状か?}
    B -- Yes --> C[全社障害の可能性]
    C --> C1[回線業者・<br/>WAN ルーター確認]
    C --> C2[情シスへ即時エスカレーション]

    B -- No --> D{有線 or 無線?}
    D -- 有線 --> E[LAN ケーブル抜き差し]
    D -- 無線 --> F[Wi-Fi OFF/ON]

    E --> G[ipconfig /all で<br/>IP 取得確認]
    F --> G

    G --> H{IP は<br/>取得できているか?}
    H -- No --> H1[DHCP 失敗<br/>release/renew 実行]
    H -- Yes --> I[ping ゲートウェイ]

    I --> J{応答あり?}
    J -- No --> J1[LAN 内の問題<br/>スイッチ・ケーブル確認]
    J -- Yes --> K[ping 8.8.8.8]

    K --> L{応答あり?}
    L -- No --> L1[インターネット出口の問題<br/>FW・ルーター確認]
    L -- Yes --> M[ping google.com]

    M --> N{応答あり?}
    N -- No --> N1[DNS の問題<br/>DNS 設定確認]
    N -- Yes --> O[ブラウザ側問題<br/>キャッシュ削除・別ブラウザ試行]
```

## 2. メール送受信ができない

```mermaid
flowchart TD
    A[ユーザー報告: メール送受信できない] --> B{送信 or 受信<br/>or 両方?}

    B -- 送信のみ --> C[エラーコード確認]
    B -- 受信のみ --> D[迷惑メールフォルダ確認]
    B -- 両方 --> E[ネットワーク接続確認]

    C --> C1{エラー内容}
    C1 -- 認証失敗 --> C2[パスワード再入力<br/>変更後の同期確認]
    C1 -- 容量超過 --> C3[クラウド共有を案内]
    C1 -- SMTP 接続 --> E

    D --> D1{該当メール<br/>あり?}
    D1 -- Yes --> D2[フィルタ条件の見直し<br/>差出人を信頼リストへ]
    D1 -- No --> D3[受信ルール確認]
    D3 --> D4{ルールで<br/>振分されている?}
    D4 -- Yes --> D5[ルール修正]
    D4 -- No --> D6[送信元にバウンス有無を確認依頼]

    E --> E1[インターネット接続 OK?]
    E1 -- No --> E2[→ ネットワークフローへ]
    E1 -- Yes --> E3[メールサーバー稼働確認<br/>情シスへ確認]
```

## 3. 印刷ができない

```mermaid
flowchart TD
    A[ユーザー報告: 印刷できない] --> B{他の社員は<br/>印刷できるか?}

    B -- No --> C[プリンタ本体・<br/>ネットワーク側の問題]
    C --> C1[プリンタの表示パネル確認]
    C1 --> C2{エラー表示}
    C2 -- 紙詰まり --> C3[紙除去・再起動]
    C2 -- トナー切れ --> C4[消耗品交換]
    C2 -- オフライン --> C5[プリンタ再起動<br/>IP 確認・ping 試行]
    C2 -- なし --> C6[ネットワーク疎通確認]

    B -- Yes --> D[当該 PC 個別の問題]
    D --> D1{印刷ジョブが<br/>溜まっているか?}
    D1 -- Yes --> D2[スプーラー再起動<br/>net stop/start spooler]
    D1 -- No --> D3[既定プリンタ確認]
    D3 --> D4{正しい<br/>プリンタか?}
    D4 -- No --> D5[既定プリンタ変更]
    D4 -- Yes --> D6[ドライバー再インストール]
```

## 4. PC の動作が遅い

```mermaid
flowchart TD
    A[ユーザー報告: PC が遅い] --> B{いつから?}

    B -- 起動直後だけ --> C[スタートアップアプリ確認]
    C --> C1[不要常駐を無効化]

    B -- 常時遅い --> D[タスクマネージャー確認]
    D --> D1{ボトルネック}

    D1 -- CPU 高負荷 --> E[使用率の高いプロセス特定]
    E --> E1{業務に必要?}
    E1 -- No --> E2[終了 or アンインストール]
    E1 -- Yes --> E3[スペック不足の可能性<br/>増設・更新検討]

    D1 -- メモリ高負荷 --> F[使用量の高いプロセス特定]
    F --> F1[不要タブ・アプリ閉じる]
    F1 --> F2{改善せず?}
    F2 -- Yes --> F3[メモリ増設見積]

    D1 -- ディスク 100% --> G[Windows Update / Defender /<br/>同期クライアントの動作確認]
    G --> G0{レガシー HDD 機?}
    G0 -- Yes --> G1[SSD 換装提案]
    G0 -- No --> G2[インデックス再構築<br/>SysMain 設定確認]

    D1 -- ネットワーク高 --> H[OneDrive 等の同期確認]
    H --> H1[同期一時停止で改善するか]

    B -- 特定アプリのみ --> I[該当アプリ再起動]
    I --> I1{改善?}
    I1 -- No --> I2[アプリ再インストール]
```

## 5. パスワード関連トラブル

```mermaid
flowchart TD
    A[ユーザー報告: ログインできない] --> B{どのシステム?}

    B -- Windows ログイン --> C[ローカル or ドメイン?]
    C -- ローカル --> C1[管理者アカウントで<br/>パスワードリセット]
    C -- ドメイン --> C2[AD でアカウント状態確認]

    C2 --> C3{ロック状態?}
    C3 -- Yes --> C4[本人確認 → ロック解除]
    C3 -- No --> C5[パスワード期限切れ確認]
    C5 --> C6{期限切れ?}
    C6 -- Yes --> C7[本人確認 → リセット]
    C6 -- No --> C8[CapsLock・キーボード配列確認]

    B -- Microsoft 365 --> D[ブラウザでログイン試行]
    D --> D1{ブラウザで<br/>成功?}
    D1 -- Yes --> D2[クライアントアプリの<br/>キャッシュ削除・再サインイン]
    D1 -- No --> D3[管理センターで<br/>アカウント状態確認]

    B -- 業務アプリ --> E[アプリ管理者へエスカレーション]

    C4 --> Z[再発防止策]
    C7 --> Z
    Z --> Z1[他端末の古い<br/>認証情報削除を案内]
    Z --> Z2[作業ログ記録]
```

## 6. 不審メール受信 / クリックしてしまった

```mermaid
flowchart TD
    A[ユーザー報告: 不審メール] --> B{まだ<br/>開いていない?}

    B -- Yes --> C[開かないよう指示]
    C --> C1[ヘッダー確認<br/>送信元ドメイン精査]
    C1 --> C2{なりすまし?}
    C2 -- Yes --> C3[情シスへ転送<br/>全社注意喚起検討]
    C2 -- No --> C4[個別判断<br/>削除指示 or 詳細確認]

    B -- No: 開いた --> D{URL クリック or<br/>添付実行?}
    D -- No --> D1[メールを閉じる<br/>情シスへ転送・削除]

    D -- URL クリックのみ --> E[即時対応]
    D -- 添付実行 --> F[緊急対応]

    E --> E1[ブラウザ閉じる]
    E1 --> E2[認証情報を入力した?]
    E2 -- Yes --> E3[該当パスワード全変更]
    E2 -- No --> E4[念のためフルスキャン]

    F --> F1[即座にネットワーク切断<br/>LAN 抜く / Wi-Fi OFF]
    F1 --> F2[シャットダウンせず<br/>情シスへ即報告]
    F2 --> F3[端末隔離 → 解析 or 再キッティング]
    F3 --> F4[同一アカウントの<br/>認証情報を全変更]
    F4 --> F5[インシデント記録・<br/>関係者へ報告]
```

## フローチャート設計の考え方

私が切り分けフローを作る際に意識した点です。

| 観点 | 内容 |
| --- | --- |
| **二分木で考える** | 「Yes / No」で枝を分け、判断を機械的にする |
| **共通の入口** | 「他の社員も同じか」「いつから」「再現性」を最初に確認 |
| **コストの低い順** | 再起動 → 再インストール → 再キッティング の順で試す |
| **エスカレーション基準** | 30 分試して進展なし or 影響範囲が広がる場合は即上位へ |
| **記録を残す** | フローのどこで解決したかをチケットに記載し、FAQ・手順書に還元 |
