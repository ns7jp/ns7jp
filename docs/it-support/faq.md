# 社内 IT サポート 想定 FAQ 集

> **本ドキュメントの位置付け**
>
> - 私が IT サポート / 社内 SE 補助業務に従事した場合、最初に整備したい **FAQ の設計サンプル** です
> - 一般的な社内 IT で頻度が高いと考えられる問い合わせを、訓練校での学習・公開資料・自身の調査をもとに整理したものであり、**特定企業での対応実績ではありません**
> - 「業務設計・手順書整備のアウトプットスキル」を示すことが目的です

社内ヘルプデスク・社内 SE 補助業務を想定し、実際の問い合わせで頻度が高いと考えられる項目を、**一次切り分け手順**とともにまとめています。

問い合わせを受けた際の目的は次の 3 点です。

1. ユーザーの業務を最短で復旧させる
2. 同じ問い合わせを繰り返させない（恒久対応・FAQ 化）
3. 障害の傾向を蓄積し、社内の改善提案につなげる

## PC・OS 関連

### Q1. PC の動作が遅い

**一次切り分け（受付者側）**

1. 「いつから」「どの操作で」遅いかを確認（特定アプリのみ / 全体）
2. 直近の Windows Update / アプリ更新の有無を確認
3. 起動直後だけ遅いか、常時遅いか

**対応手順**

| 確認項目 | コマンド・操作 | 判断基準 |
| --- | --- | --- |
| CPU 使用率 | タスクマネージャー → パフォーマンス | 80% 以上が継続するプロセスを特定 |
| メモリ使用率 | 同上 | 物理メモリ 90% 以上は要対応 |
| ディスク使用率 | 同上 | 100% 張り付きは HDD 寿命または常駐の暴走 |
| 起動アプリ | タスクマネージャー → スタートアップ | 不要アプリを「無効」 |
| ディスク空き | エクスプローラー → C ドライブ | 10% 未満なら一時ファイル削除 |

**恒久対応の例**

- 起動時の常駐アプリを棚卸し
- HDD → SSD 換装の提案（情報システム部判断）
- メモリ増設の見積もり提示

---

### Q2. Windows Update 後に PC が起動しない

**一次切り分け**

1. メーカー名・モデル名を聞き取り
2. エラーコードや画面の状態を撮影してもらう（スマホ可）
3. 業務影響の緊急度確認（即対応 / 翌日対応）

**対応手順**

1. セーフモード起動を試行（電源強制断 3 回 → 自動修復メニュー）
2. 「更新プログラムのアンインストール」で直近の更新を削除
3. 改善しない場合はシステムの復元
4. それでも改善しなければイメージからの再キッティング

## ネットワーク関連

### Q3. インターネットに繋がらない

**一次切り分けフロー**

```text
1. 他の社員も同様か？  ─ Yes → 全社障害の可能性 → ネットワーク機器・回線業者確認
                       └ No  → 当該 PC 個別の問題
                                ↓
2. 有線 / 無線 どちらか？
                                ↓
3. ping で切り分け（下表）
```

**ping による切り分け表**

| 宛先 | 成功 / 失敗 | 判断 |
| --- | --- | --- |
| `127.0.0.1`（自分） | 失敗 | TCP/IP スタック異常 → ネットワークアダプター再インストール |
| デフォルトゲートウェイ | 失敗 | LAN 区間の問題 → ケーブル・スイッチ・IP 設定確認 |
| `8.8.8.8` | 失敗 | インターネット出口の問題 → ファイアウォール・ルーター確認 |
| `google.com` | 失敗（IP は成功） | DNS の問題 → DNS サーバー設定確認 |

**よくある原因**

- 社内 Wi-Fi のパスワード変更後、PC に古い情報が残っている
- DHCP リース切れ（`ipconfig /release` → `ipconfig /renew`）
- プロキシ設定の自動構成が失敗（社内プロキシ環境の場合）
- VPN 切断後に DNS 設定が戻っていない

---

### Q4. 社内 VPN に繋がらない

**一次切り分け**

| 確認項目 | 内容 |
| --- | --- |
| インターネット接続 | VPN 以外の通信は可能か |
| クライアント版本 | 配布バージョンと一致しているか |
| 認証情報 | パスワード変更直後でないか（社内 AD 連動の場合） |
| 接続先 | 接続先サーバー / プロファイル名は正しいか |
| 時刻同期 | PC 時刻が大幅にずれていないか（証明書認証で失敗の原因） |

**対応手順**

1. VPN クライアントの再起動 → PC の再起動
2. 「接続先プロファイル」を一度削除して再追加
3. クライアントの再インストール（バージョン更新含む）
4. 解決しなければ情報システム部にエスカレーション（ログを添付）

## メール・コミュニケーション関連

### Q5. メールが送れない / 届かない

**送れない場合の切り分け**

| エラー内容 | 想定原因 | 対応 |
| --- | --- | --- |
| 添付ファイル容量超過 | サーバー上限 | クラウドストレージ共有を案内 |
| 認証エラー | パスワード変更未反映 | クライアント側パスワード再入力 |
| SMTP 接続エラー | ネットワーク / プロキシ | ネットワーク切り分けへ |
| 宛先不明（554 等） | 宛先タイプミス / 相手側問題 | 宛先確認、相手側担当者へ連絡依頼 |

**届かない場合の切り分け**

1. 迷惑メールフォルダを確認
2. 受信ルール（仕分けルール）の誤動作を確認
3. メーリングリスト・配信停止の状態を確認
4. 送信者にバウンスメールが返っていないか確認依頼

## アカウント・認証関連

### Q6. パスワードを忘れた

**一次切り分け**

- 対象システム（AD / Microsoft 365 / 業務アプリ / VPN 等）の特定
- 本人確認手順（社員番号・所属・上長承認の要否）の遵守

**対応フロー**

```text
本人確認 OK
   ↓
パスワード初期化（情シス権限で実行）
   ↓
初期パスワードを「対面」または「社内チャット DM」で連携
（電子メールでの平文送付は NG）
   ↓
ユーザー初回ログイン時に変更を強制
   ↓
作業ログをチケットに記録
```

**注意点**

- 本人確認をスキップした初期化は **絶対に行わない**（ソーシャルエンジニアリング対策）
- 上長承認のないリセットは事後にトラブルになる
- 初期パスワードは「初回ログイン時変更」を必ず強制

---

### Q7. アカウントがロックされた

**一次切り分け**

| 確認項目 | 内容 |
| --- | --- |
| 連続失敗回数 | AD のポリシー上限を超えたか |
| 端末を変えても発生するか | ID/PW の問題か、端末の問題か |
| パスワード変更直後か | 古いパスワードが他端末・他アプリに残っている可能性 |

**よくある原因**

- スマホのメールアプリに古いパスワードが残ったまま、サーバーへリトライし続けてロック
- 自宅 PC で会社メールを設定したまま放置
- 共有 PC で他のユーザーが誤って ID を入力

**対応**

1. アカウントロック解除
2. パスワード変更後に他端末（スマホ・タブレット）の再設定を案内
3. ロック発生時刻のイベントログから発生元 IP を特定し、再発防止

## 周辺機器・印刷関連

### Q8. 印刷ができない

**切り分けフロー**

```text
1. 他の人は同じプリンタに印刷できるか？
    Yes → 当該 PC の設定 / ドライバー問題
    No  → プリンタ本体 / ネットワークの問題
            ↓
2. プリンタ本体のエラー表示は？（紙詰まり / トナー切れ / オフライン）
            ↓
3. ping でプリンタの IP に疎通確認
```

**よくある原因と対応**

| 症状 | 原因 | 対応 |
| --- | --- | --- |
| 「オフライン」表示 | プリンタの再起動忘れ / IP 変更 | 電源 OFF/ON、IP 再設定 |
| 印刷ジョブが詰まる | ドライバー異常 | スプーラー再起動（`net stop spooler && net start spooler`） |
| 文字化け | ドライバー不一致 | 正しいドライバーを再インストール |
| 両面・カラー設定不可 | 機種制限 / 権限設定 | 印刷設定の既定値を確認 |

---

### Q9. 外部モニターが映らない

**一次切り分け**

| 確認 | 内容 |
| --- | --- |
| ケーブル種別 | HDMI / DisplayPort / USB-C のどれか |
| モニター入力切替 | 正しい入力ソースが選ばれているか |
| ケーブル交換 | 別のケーブルで再現するか |
| 別 PC で再現 | 別 PC を繋いで映るか |
| ディスプレイ設定 | `Win + P` で「拡張」になっているか |

## セキュリティ・マルウェア関連

### Q10. 不審なメールが届いた / 開いてしまった

**開く前（相談を受けた）の対応**

1. メールを開かず、ヘッダー情報を確認（送信元ドメインのなりすまし）
2. URL は実機でクリックせず、テキストとして確認
3. 添付ファイルは開かない
4. 情シスへ転送（生メールを添付）→ 削除指示

**開いてしまった場合**

```text
1. PC を即座にネットワークから切り離す（LAN ケーブル抜去 / Wi-Fi OFF）
   ※ シャットダウンは メモリ証跡が消えるため 情シス判断を仰ぐ
2. 情シス / 情報セキュリティ責任者へ即時報告
3. 開いた時刻・操作内容・表示された画面を記録
4. 当該 PC を隔離した状態で、フルスキャン or 再キッティング
5. 同一アカウントの認証情報（メール / VPN / 業務システム）を全変更
```

**ユーザー教育の機会として**

- 「送信元アドレスのドメインを確認する」
- 「文中の URL にマウスオーバーして実 URL を見る」
- 「不安なら開く前に情シスへ転送」
の 3 点を全社向け案内に反映。

## 受付時の共通テンプレート

問い合わせ受付時、以下を必ず確認することで切り分けが速くなります。

```text
■ 発生日時：
■ 利用者氏名・所属：
■ 端末（PC 名 / 資産番号）：
■ OS / アプリバージョン：
■ 事象（何をしたら何が起きたか）：
■ エラーメッセージ（スクリーンショット）：
■ 再現性（毎回 / たまに / 1 回だけ）：
■ 業務影響度（緊急 / 高 / 中 / 低）：
■ 自分で試したこと：
```
